1.4.4. Cliente/Servidor

Cliente/Servidor

La arquitectura cliente-servidor es una de las más populares en la construcción de aplicaciones en la actualidad. Básicamente se llama así a toda arquitectura en la que participan dos componentes: uno es el cliente que utiliza unos servicios, y otro es el servidor que proporciona esos servicios. Entre ambos se tiene que efectuar una comunicación de red, habitualmente mediante Internet.

La clasificación de los lenguajes de programación desde el punto de vista cliente/Servidor es adecuada para cuando sea necesaria la programación de aplicaciones Cliente/Servidor en la que por una parte habrá una aplicación encargada de servir las peticiones procedentes de aplicaciones clientes. Un ejemplo de este tipo de arquitectura Cliente/Servidor lo tenemos en la mayoría de aplicaciones web en las que un programa servidor, ubicado en un servidor web, tendrá la función de servir las peticiones de cada una de las aplicaciones web clientes que se ejecuten desde cada uno de los navegadores web.

Lenguajes del lado del cliente

Los lenguajes de programación de cliente serán aquellos que en un entorno Cliente/Servidor, se interpretan y ejecutan siempre desde el ordenador cliente. En programación web esta ejecución e interpretación siempre tendrá lugar desde cada uno de los navegadores webs de cada uno de los ordenadores clientes que interactúan con el Servidor web.

Como ejemplos de lenguajes de programación de cliente tenemos el Javascript y el VBscript.

lenguajes del lado del Servidor

Los lenguajes de Servidor son aquellos utilizados en aplicaciones cliente/servidor para el desarrollo de la parte correspondiente al Servidor. La ejecución de las aplicaciones desarrolladas con estos lenguajes se restringe exc lusivamente al servidor.

Ejemplos de lenguajes de servidor son: Java, PHP, Perl,ASPm nodejs.

Java

Los programas Java se pueden escribir como applets, servlets o aplicaciones.

Los applets de Java son pequeños programas diseñados para agregar interactividad a una página web. Se inician mediante un navegador de Internet; no pueden funcionar de forma independiente. Cuando el usuario solicita una página web que contiene un applet, éste se descarga  en el ordenador del usuario y es ejecutado por la JVM en el navegador. Los applets se ejecutan en la parte del cliente.

Los servlets Java son invocados por el servidor, donde se ejecutan, sin descargarse al cliente. Normalmente, los servlets generan contenido web dinámicamente,  leyendo y escribiendo en una base de datos utilizando JDBC (Java Data- base Connectivity).

Las aplicaciones Java se ejecutan de forma independiente en un ordenador, al igual que un progrma C, C++, Pascal, etc.
