2. Representació de Dades

Nombres Binaris

La CPU només entén nombres binaris, els dígits dels quals són 0 o 1. Totes les dades s'emmagatzemen en la memòria d'un ordinador com a Dígits binaris. Un bit conté un dígit binari. Un byte conté vuit dígits binaris. Els nombres binaris s'expressen en el sistema de base 2, perquè només hi ha 2 valors en aquest sistema, 0 i 1. Per contra, la majoria de la gent està acostumada al sistema decimal, o de base 10, que utilitza els valors del 0 al 9.

Existeixen altres sistemes de numeració, com el sistema octal, o de base 8, que utilitza els dígits del 0 al 7 i el sistema hexadecimal, o de base 16, que utilitza els dígits del 0 al 9 i les lletres de la A a la F.

Tal com ho coneixem en el sistema decimal, el nombre 359 està compost pels següents tres dígits:

  • 3, representant les centenes, o 102
  • 5, representant les desenes, o 101
  • 9, representant les unitats, o 100

Per tant, podem escriure 359 com

359 = 3 * 102 + 5 * 101 + 9 * 100

Així, el nombre decimal 359 s'escriu com una combinació lineal de potències de 10 amb coeficients de l'alfabet base 10, és a dir, els dígits del 0 al 9.

De manera similar, el nombre binari 11011 s'escriu com una combinació lineal de potències de 2 amb coeficients de l'alfabet base 2, és a dir, els dígits 0 i 1.

Per exemple, el nombre binari 11011 es pot escriure com

11011 = 1*24 + 1*23 + 0*22 + 1*21 + 1*20

La Taula següent enumera els equivalents binaris per als nombres decimals del 0 al 8 i enumera els equivalents decimals de les primeres 15 potències de 2.

Observa que a mesura que comptem en increments d'1 (en decimal), l'últim dígit (en binari) alterna entre 0 i 1.

Per convertir un nombre binari a decimal, multiplica cada dígit del nombre binari per 2posició - 1, considerant la posició més a la dreta com a posició 1 i augmentant en desplaçar-nos cap a l'esquerra. Després, suma els productes.

Usant aquest mètode, calculem l'equivalent del nombre binari 11010 en el nostre sistema decimal.

11010 = 1*24 + 1*23 + 0*22 + 1*21 + 0*20
      =  16  +  8   +  0  +   2  +  0
      =     26

Ara examinem com convertir un nombre decimal en un nombre binari.

Convertim el nombre decimal 359 al seu equivalent binari. Un mètode consisteix a sumar les potències de 2 necessàries per a arribar a 359. L'estratègia a seguir consisteix en:

  1. trobar la major potència de 2 que siga menor o igual que el nombre decimal.
  2. Restar aquest nombre al nombre decimal,
  3. Repetir el procés amb  el residu, i així successivament, fins a arribar a 0.

(1) La major potència de 2 menor que 359 és 256, o 28 (la següent potència major de 2 seria 512, que és major que 359).

(2) Restar 256 de 359 ens dóna 103 (359 - 256 = 103), per la qual cosa ara tenim 359 = 28 * 1 + 103

(3) (1) Ara apliquem el mateix procediment a 103. La major potència de 2 que és menor que 103 és 64, o 26. Això vol dir que no hi ha cap factor per a 27, per la qual cosa el valor d'aquest dígit és 0.

(2) Restar 64 de 103 ens dóna 39. Ara tenim

359 = 28 * 1 + 27 * 0 + 26 * 1 + 39

(3)(1)(2) Repetint el procediment per a 39, trobem que la major potència de 2 menor que 39 és 32 o 25. Restar 32 de 39 ens dóna 7.

Així que ara tenim:      359 = 28 * 1 + 27 * 0 + 26 * 1 + 25 * 1 + 7

(3)(1)(2) Repetint el procediment per a 7, la major potència de 2 menor que 7 és 22, o 4. Això vol dir que no hi ha factors per a 24 o 23, per la qual cosa el valor de cadascun d'aquests dígits són 0. Restar 4 de 7 ens dóna 3, per la qual cosa tenim:

359 = 28 * 1 + 2⁷ * 0 + 2⁶ * 1 + 2⁵ * 1 + 2⁴ * 0 + 2³ * 0 + 2² * 1 + 3

Repetint el procediment per a 3, la major potència de 2 menor que 3 és 2, o 2¹, i tenim:

359 = 28 * 1 + 2⁷ * 0 + 2⁶ * 1 + 2⁵ * 1 + 2⁴ * 0 + 2³ * 0 + 2² * 1 + 2¹ * 1 + 1

1 és una potència de 2; és 2⁰, per la qual cosa finalment tenim

359 = 28 * 1 + 2⁷ * 0 + 2⁶ * 1 + 2⁵ * 1 + 2⁴ * 0 + 2³ * 0 + 2² * 1 + 2¹ * 1 + 2⁰ * 1

359 es pot representar en el sistema binari com:

359  =  1  0  1  1  0  0  1  1  1  
o bé          1 0110 0111

Ús de números hexadecimals per a representar nombres binaris

Com pots veure, els nombres binaris poden arribar a ser bastant llargs. Amb només dos valors possibles, 0 i 1, es necessiten 16 dígits binaris per a representar el 32.768. Per aquesta raó, el sistema hexadecimal, o de base 16, s'utilitza sovint com a representació abreujada de nombres binaris. El sistema hexadecimal utilitza 16 dígits: del 0 al 9 i de la A a la F. Les lletres de la A a la F representen els valors 10, 11, 12, 13, 14 i 15.

El valor màxim que es pot representar en quatre dígits binaris és 2⁴ - 1, o 15. El valor màxim d'un dígit hexadecimal també és 15, que es representa amb la lletra F. Per tant, pots reduir la mida d'un nombre binari utilitzant dígits hexadecimals per a representar cada grup de quatre dígits binaris.

Per a representar un nombre binari en hexadecimal, simplement substitueix el dígit hexadecimal apropiat per a cada conjunt de quatre dígits binaris.

0001 1010 1111 1001 1011 0011 1011 1110
 1    A    F    9    B    3    B    E

Representació de caràcters amb el conjunt de caràcters Unicode

Java representa caràcters mitjançant l'Estàndard Mundial de Caràcters Unicode, o simplement Unicode. Cada caràcter Unicode es representa amb 16 bits o dos bytes. Això vol dir que el conjunt de caràcters Unicode pot codificar 65.536 caràcters.

L'objectiu del consorci Unicode és oferir compatibilitat amb un conjunt de caràcters internacionals, incloent els caràcters imprimibles del teclat QWERTY estàndard, així com caràcters internacionals com é o λ.

Molts llenguatges de programació emmagatzemen caràcters utilitzant el conjunt de caràcters ASCII (Codi estàndard americà per a l'intercanvi d'informació), que utilitza 7 bits per a codificar cada caràcter i, per tant, només pot representar 128 caràcters. Per a ser compatibles amb el conjunt de caràcters ASCII, els primers 128 caràcters del conjunt de
caràcters Unicode són els mateixos que els del conjunt de caràcters ASCII.

Per a obtindre més informació sobre el conjunt de caràcters Unicode, consulte l'Apèndix C o visite el lloc web del Consorci Unicode a http://www.Unicode.org.
