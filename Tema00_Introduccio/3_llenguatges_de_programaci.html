3. Llenguatges de programació

Llenguatges d'alt i baix nivell

Els llenguatges de programació es poden classificar en tres tipus:

■ llenguatge de màquina
■ llenguatge assemblador
■ llenguatge d'alt nivell

En els primers dies de la informàtica, els programadors solien utilitzar llenguatge de màquina o llenguatge assemblador. El llenguatge de màquina utilitza codis binaris o cadenes de zeros i uns, per a executar el conjunt d'instruccions de la CPU i fer referència a la memòria d'adreces. Aquest mètode de programació és extremadament desafiant i requereix molt de temps. A més, el codi escrit en llenguatge de màquina no és portable a altres arquitectures informàtiques.

No obstant això, avui en dia rarament utilitzen el llenguatge de màquina. Els llenguatges assembladors estan un pas per sobre del llenguatge de màquina, utilitzant símbols, noms per a adreces de memòria i mnemonics per a instruccions del processador.

Igual que el llenguatge de màquina, els llenguatges assembladors també depenen de la CPU i no són portables entre ordinadors amb diferents processadors (per exemple, entre Intel i SPARC). El llenguatge assemblador és més fàcil d'escriure que el llenguatge de màquina, però encara requereix un esforç considerable, per la qual cosa sol usar-se només quan el programa requereix característiques, com l'accés directe al maquinari, que no són compatibles amb un llenguatge d'alt nivell.

Els llenguatges d'alt nivell, com Fortran, Pascal, Perl, Objective­C, PHP, C++, Python i Java, s'assemblen més a l'anglès que al llenguatge de màquina, cosa que els fa molt més fàcils d'usar per al desenvolupament de programari i més portables entre arquitectures de CPU. Per aquesta raó, els programadors han adoptat llenguatges d'alt nivell per a cada vegada més aplicacions.

Els llenguatges d'alt nivell es compilen, s'interpreten o una combinació de tots dos. Un programa escrit en un llenguatge compilat, com C++, és convertit per un compilador en codi de màquina, i després el codi de màquina s'executa.

En canvi, un programa escrit en un llenguatge interpretat, com Perl, es llegeix i es converteix a codi de màquina, línia a línia, en temps d'execució. Normalment, un programa escrit en un llenguatge interpretat s'executa més lentament que el seu equivalent escrit en un llenguatge compilat.

Java utilitza una combinació de compilador i intèrpret. Un programa Java es compila primer en codis de bytes independents del processador; posteriorment, el fitxer de codi de bytes s'interpreta en temps d'execució mitjançant un programari anomenat Màquina Virtual de Java (JVM).

Introducció a la programació orientada a objectes

Els llenguatges inicials d'alt nivell, com Fortran o Pascal, eren procedimentals. Normalment, els programadors escrivien codi específic per a cada tasca en procediments o funcions independents, i els invocaven des d'altres seccions del programa per a realitzar diverses tasques. Les dades del programa generalment es compartien entre els procediments.

A mitjans de la dècada de 1970, es va introduir Smalltalk, el primer llenguatge de programació orientat a objectes, que permetia als programadors escriure codi amb un enfocament diferent. Mentre que els procediments o funcions s'ocupaven principalment de tipus de dades bàsics com enters, nombres reals o caràcters individuals, Smalltalk va proporcionar al programador una nova eina: les classes i els objectes d'aquestes classes.

Una classe permet al programador encapsular dades i les funcions necessàries per a manipular-les en un sol paquet. Una classe defineix essencialment una plantilla, o model, a partir del qual es creen els objectes. La creació d'un objecte es denomina instanciació. Per tant, els objectes es creen (instancien) segons el disseny de la classe.

Una classe podria representar alguna cosa real, com una persona. La classe podria tenir diversos atributs, com, en l'exemple de la classe "persona", un nom, un cognom i una edat. La classe també proporcionaria codi, anomenat mètodes, que permet al creador de l'objecte establir i recuperar els valors dels atributs.

Un gran avantatge de la programació orientada a objectes és que les classes ben escrites poden ser reutilitzades per nous programes, reduint així el temps de desenvolupament futur.

El llenguatge Java

23 de maig de 1995, Sun Microsystems va presentar Java, originalment anomenat Oak, com un llenguatge lliure i orientat a objectes, dirigit a aplicacions integrades per a dispositius de consum. Immediatament es va incorporar una màquina virtual de Java que es va integrar en el navegador d'Internet Netscape Navigator, i a mesura que Internet creixia, petits programes Java, coneguts com a applets, van començar a aparèixer en les pàgines web en quantitats cada vegada majors. La sintaxi de Java és bàsicament idèntica (amb algunes petites excepcions) a la de C++, i aviat programadors de tot el món van començar a adonar-se dels avantatges d'usar Java. Aquests avantatges inclouen:

  • sintaxi idèntica a la de C++
  • orientació a objectes
  • característiques relacionades amb Internet, com a subprogrames, que són executats pel navegador, i servlets, que són executats pel servidor web
  • una amplia biblioteca de classes que poden reutilitzar-se fàcilment, incloent Classes Swing (proveir una interfície gràfica d'usuari) i Java Data-base Connectivity (JDBC) per a comunicar-se amb una base de dades
  • portabilitat entre totes les plataformes que admeten una Màquina Virtual Java
  • Xarxes integrades
  • Disponibilitat de codi obert del Kit de desenvolupament de Java

Com esmentem anteriorment, un programa Java es compila primer en codis de bytes independents del processador, que després la Màquina Virtual de Java (JVM) interpreta en temps d'execució. Com el seu nom indica, la JVM simula un processador virtual amb el seu propi conjunt d'instruccions, registres i punter d'instruccions. Per tant, per a executar un programa Java, només es necessita una JVM. Afortunadament, les JVM estan disponibles en totes les principals plataformes informàtiques.

Els programes Java es poden escriure com a applets, servlets o aplicacions.

Els applets són petits programes dissenyats per a afegir interactivitat a una pàgina web. S'inicien mitjançant un navegador d'Internet; no poden executar-se de forma independent. Quan l'usuari sol·licita una pàgina web que fa servir un applet, aquest es descarrega en el seu ordinador i la JVM del navegador l'executa. No obstant això, a causa d'incompatibilitats dels navegadors, limitacions imposades per les funcions de seguretat i temps de descàrrega lents, els applets han caigut en desús.

Els servlets Java són invocats pel servidor web i s'executen en ell, sense ser descarregats al client. Normalment, els servlets generen contingut web dinàmicament llegint i escrivint en una base de dades mitjançant JDBC (Connectivitat de Base de Dades Java).

Les aplicacions Java s'executen de forma independent en un equip client.

Oracle Corporation, que va adquirir Sun Microsystems en gener de 2010, ofereix un valuós lloc web sobre Java (https://www.oracle.com/es/java/), que conté informació sobre l'ús de les classes, un tutorial sobre Java i molts altres recursos per al programador.
