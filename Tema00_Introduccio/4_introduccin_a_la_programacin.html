4. Introducción a la programación

Fundamentos de programación

En muchos sentidos, programar es como resolver un rompecabezas. Tienes una tarea que realizar y conoces las operaciones que puede realizar una computadora (entrada, cálculos, comparaciones, reorganización de elementos y salida). Como programador, tu trabajo consiste en descomponer una tarea en pasos individuales y ordenados de entrada, cálculo, comparación, reorganización y salida.

Por ejemplo, supongamos un programa para hallar la suma de dos números. Primero, el programa debe leer (introducir) los números al ordenador. Después, debe sumar los dos números (calcular). Finalmente, debe escribir (mostrar) la suma.

Observe que este programa consta de pasos, llamados instrucciones, que se ejecutan en orden ("Primero", "Después", "Finalmente"). Realizar operaciones en orden, una tras otra,
se denomina procesamiento secuencial.

El orden en que la computadora ejecuta las instrucciones es crucial en la programación. No se puede calcular la suma de dos números sin haberlos leído, ni se puede mostrar el resultado de una suma sin haberla calculado. Por lo tanto, la programación requiere que el programador especifique el orden de las instrucciones, lo que se denomina flujo de control del programa.

Existen cuatro maneras diferentes en que el flujo de control puede progresar a través de un programa:

  1. ejecución secuencial,
  2. llamada a método,
  3. selección y
  4. bucle.

Debido a que obtener el flujo de control correcto es esencial para que un programa produzca el resultado correcto, los programadores utilizan una herramienta llamada pseudocódigo para ayudarlos a diseñar el flujo de control antes de escribir el código.

Diseño de programas con pseudocódigo

El pseudocódigo es un método para expresar el orden de las instrucciones de un programa en un idioma más parecido al coloquial, en lugar de un lenguaje de programación. De esta forma, el programador puede concentrarse en el diseño de un programa sin complicarse con la sintaxis del lenguaje de programación.

El pseudocódigo para calcular la suma de dos números podría ser:

inicio
    leer primer_número
    leer segundo_número
    calcular  total <-- primer_numero + segundo_numero
    mostrar total
fin

Desarrollo de una aplicación Java

Escribir una aplicación Java consta de varios pasos:

  • escribir el código,
  • compilarlo y
  • ejecutarlo.

El código fuente de Java se almacena en un archivo de texto con la extensión .java. La compilación del código crea uno o más archivos .class , que contienen códigos de bytes independientes del procesador.

La Máquina Virtual Java (JVM) traduce los códigos de bytes en instrucciones a nivel de máquina para el procesador en el que se ejecuta la aplicación Java.

Oracle ofrece un Java SE Development Toolkit (JDK) en su sitio web  que se puede descargar de forma gratuita. El JDK contiene un compilador, una JVM y un visor de applets, que es un navegador minimalista. Además, el JDK contiene una amplia gama de clases Java preescritas que los programadores pueden usar en sus aplicaciones Java.

Para compilar el código, invoque el compilador desde la línea de comandos:

javac  NombreClase.java

donde ClassName.java es el nombre del archivo fuente. Si su programa, escrito en el archivo ClassName.java, se compila correctamente, se creará un nuevo
archivo, ClassName.class, en su directorio actual.
