3. Lenguajes de programación

Lenguajes de alto y bajo nivel

Los lenguajes de programación se pueden clasificar en tres tipos:

■ lenguaje de máquina
■ lenguaje ensamblador
■ lenguaje de alto nivel

En los primeros días de la informática, los programadores solían utilizar lenguaje máquina o lenguaje ensamblador. El lenguaje máquina utiliza códigos binarios o cadenas de ceros y unos, para ejecutar el conjunto de instrucciones de la CPU y hacer referencia a la memoria de direcciones. Este método de programación es extremadamente desafiante y requiere mucho tiempo.  Además, el código escrito en lenguaje máquina no es portable a otras arquitecturas informáticas.

Sin embargo, hoy en día rara vez utilizan el lenguaje máquina. Los lenguajes ensambladores están un paso por encima del lenguaje máquina, utilizando símbolos, nombres para direcciones de memoria y mnemónicos para instrucciones del procesador.

Al igual que el lenguaje máquina, los lenguajes ensambladores también dependen de la CPU y no son portables entre computadoras con diferentes procesadores (por ejemplo, entre Intel y SPARC). El lenguaje ensamblador es más fácil de escribir que el lenguaje máquina, pero aun así requiere un esfuerzo considerable, por lo que suele usarse solo cuando el programa requiere características, como el acceso directo al hardware, que no son compatibles con un lenguaje de alto nivel.

Los lenguajes de alto nivel, como Fortran, Pascal, Perl, Objective­C, PHP, C++, Python y Java, se asemejan más al inglés que al lenguaje máquina, lo que los hace mucho más fáciles de usar para el desarrollo de software y más portables entre arquitecturas de CPU. Por esta razón, los programadores han adoptado lenguajes de alto nivel para cada vez más aplicaciones.

Los lenguajes de alto nivel se compilan, se interpretan o una combinación de ambos. Un programa escrito en un lenguaje compilado, como C++, es convertido por un compilador en
código máquina, y luego el código máquina se ejecuta.

En cambio, un programa escrito en un lenguaje interpretado, como Perl, se lee y se convierte a código máquina, línea a línea, en tiempo de ejecución. Normalmente, un programa escrito en un lenguaje interpretado se ejecuta más lentamente que su equivalente escrito en un lenguaje compilado.

Java utiliza una combinación de compilador e intérprete. Un programa Java se compila primero en códigos de bytes independientes del procesador; posteriormente, el archivo de código de bytes se interpreta en tiempo de ejecución mediante un software llamado Máquina Virtual de Java (JVM).

Introducción a la programación orientada a objetos

Los lenguajes iniciales de alto nivel, como Fortran o Pascal, eran procedimentales. Normalmente, los programadores escribían código específico para cada tarea en procedimientos o funciones independientes, y los invocaban desde otras secciones del programa para realizar diversas tareas. Los datos del programa generalmente se compartían entre los procedimientos.

A mediados de la década de 1970, se introdujo Smalltalk, el primer lenguaje de programación orientado a objetos, que permitía a los programadores escribir código con un enfoque diferente. Mientras que los procedimientos o funciones se ocupaban principalmente de tipos de datos básicos como enteros, números reales o caracteres individuales, Smalltalk proporcionó al programador una nueva herramienta: las clases y los objetos de esas clases.

Una clase permite al programador encapsular datos y las funciones necesarias para manipularlos en un solo paquete. Una clase define esencialmente una plantilla, o modelo, a partir del cual se
crean los objetos. La creación de un objeto se denomina instanciación. Por lo tanto, los objetos se crean (instancian) según el diseño de la clase.

Una clase podría representar algo real, como una persona. La clase podría tener varios atributos, como, en el ejemplo de la clase "persona", un nombre, un apellido y una edad. La clase también proporcionaría código, llamado métodos, que permite al creador del objeto establecer y recuperar los valores de los atributos.

Una gran ventaja de la programación orientada a objetos es que las clases bien escritas pueden ser reutilizadas por nuevos programas, reduciendo así el tiempo de desarrollo futuro.

El lenguaje Java

23 de mayo de 1995, Sun Microsystems presentó Java, originalmente llamado Oak, como un lenguaje libre y orientado a objetos, dirigido a aplicaciones integradas para dispositivos de consumo. Inmediatamente se incorporó una máquina virtual de Java que se integró en el navegador de Internet Netscape Navigator, y a medida que Internet crecía, pequeños programas Java, conocidos como applets, comenzaron a aparecer en las páginas web en cantidades cada vez mayores. La sintaxis de Java es básicamente idéntica
(con algunas pequeñas excepciones) a la de C++, y pronto programadores de todo el mundo comenzaron a darse cuenta de las ventajas de usar Java. Estas ventajas incluyen:

  • sintaxis idéntica a la de C++
  • orientación a objetos
  • características relacionadas con Internet, como subprogramas, que son ejecutados por el navegador, y servlets, que son ejecutados por el servidor web
  • una amplia biblioteca de clases que pueden reutilizarse fácilmente, incluyendo Clases Swing (proporcionar una interfaz gráfica de usuario) y  Java Data-
    base Connectivity (JDBC) para comunicarse con una base de datos
  • portabilidad entre todas las plataformas que admiten una Máquina Virtual Java
  • Redes integradas
  • Disponibilidad de código abierto del Kit de desarrollo de Java

Como mencionamos anteriormente, un programa Java se compila primero en códigos de bytes independientes del procesador, que luego la Máquina Virtual de Java (JVM) interpreta en tiempo de ejecución. Como su nombre indica, la JVM simula un procesador virtual con su propio conjunto de instrucciones, registros y puntero de instrucciones. Por lo tanto, para ejecutar un programa Java, solo se necesita una JVM. Afortunadamente, las JVM están disponibles en todas las principales plataformas informáticas.

Los programas Java se pueden escribir como applets, servlets o aplicaciones.

Los applets son pequeños programas diseñados para agregar interactividad a una página web. Se inician mediante un navegador de Internet; no pueden ejecutarse de forma independiente. Cuando el usuario solicita una página web que usa un applet, este se descarga en su ordenador y la JVM del navegador lo ejecuta. Sin embargo, debido a incompatibilidades de los navegadores, limitaciones impuestas por las funciones de seguridad y tiempos de descarga lentos, los applets han caído en desuso.

Los servlets Java son invocados por el servidor web y se ejecutan en él, sin ser descargados al cliente. Normalmente, los servlets generan contenido web dinámicamente leyendo y
escribiendo en una base de datos mediante JDBC (Conectividad de Base de Datos Java).

Las aplicaciones Java se ejecutan de forma independiente en un equipo cliente.

Oracle Corporation, que adquirió Sun Microsystems en enero de 2010, ofrece un valioso sitio web sobre Java (https://www.oracle.com/es/java/), que contiene información sobre
el uso de las clases, un tutorial sobre Java y muchos otros recursos para el programador.
