4. Introducció a la programació

Fonaments de programació

En molts sentits, programar és com resoldre un trencaclosques. Tens una tasca a realitzar i coneixes les operacions que pot realitzar un ordinador (entrada, càlculs, comparacions, reorganització d'elements i eixida). Com a programador, la teua feina consisteix a descompondre una tasca en passos individuals i ordenats d'entrada, càlcul, comparació, reorganització i eixida.

Per exemple, suposem un programa per a trobar la suma de dos nombres. Primer, el programa ha de llegir (introduir) els nombres a l'ordinador. Després, ha de sumar els dos nombres (calcular). Finalment, ha d'escriure (mostrar) la suma.

Observa que aquest programa consta de passos, anomenats instruccions, que s'executen en ordre ("Primer", "Després", "Finalment"). Realitzar operacions en ordre, una després d'una, es denomina processament seqüencial.

L'ordre en què l'ordinador executa les instruccions és crucial en la programació. No es pot calcular la suma de dos nombres sense haver-los llegit, ni es pot mostrar el resultat d'una suma sense haver-la calculat. Per tant, la programació requereix que el programador especifique l'ordre de les instruccions, el que es denomina flux de control del programa.

Existeixen quatre maneres diferents en què el flux de control pot progressar a través d'un programa:

  1. execució seqüencial,
  2. crida a mètode,
  3. selecció i
  4. bucle.

Degut a què obtindre el flux de control correcte és essencial per a què un programa produïsca el resultat correcte, els programadors utilitzen una eina anomenada pseudocodi per a ajudar-los a dissenyar el flux de control abans d'escriure el codi.

Disseny de programes amb pseudocodi

El pseudocodi és un mètode per a expressar l'ordre de les instruccions d'un programa en un idioma més semblant al col·loquial, en lloc d'un llenguatge de programació. D'aquesta forma, el programador pot concentrar-se en el disseny d'un programa sense complicar-se amb la sintaxi del llenguatge de programació.

El pseudocodi per a calcular la suma de dos nombres podria ser:

inici
    llegir primer_nombre
    llegir segon_nombre
    calcular total <-- primer_nombre + segon_nombre
    mostrar total
fi

Desenvolupament d'una aplicació Java

Escriure una aplicació Java consta de diversos passos:

  • escriure el codi,
  • compilar-lo i
  • executar-lo.

El codi font de Java s'emmagatzema en un fitxer de text amb l'extensió .java. La compilació del codi crea un o més fitxers .class, que contenen codis de bytes independents del processador.

La Màquina Virtual Java (JVM) tradueix els codis de bytes en instruccions a nivell de màquina per al processador en el qual s'executa l'aplicació Java.

Oracle ofereix un Java SE Development Toolkit (JDK) en el seu lloc web que es pot descarregar de forma gratuïta. El JDK conté un compilador, una JVM i un visor d'applets, que és un navegador minimalista. A més, el JDK conté una àmplia gamma de classes Java preescrites que els programadors poden usar en les seues aplicacions Java.

Per a compilar el codi, invoqueu el compilador des de la línia d'ordres:

javac NomClasse.java

on NomClasse.java és el nom del fitxer font. Si el vostre programa, escrit en el fitxer NomClasse.java, es compila correctament, es crearà un nou fitxer, NomClasse.class, en el vostre directori actual.
