2.3. Conversión de tipos

Es muy frecuente que en una expresión haya operandos de distintos tipos. Para evitar errores hay que convertir los operandos al mismo tipo de datos. Esta conversión se denomina casting. Existen varias formas de realizar la conversión.

Conversión implícita.

Es el compilador el que realiza la conversión de forma automática. Intenta convertir (promover) los operandos de menor precisión al de mayor precisión. Las promociones se realizan utilizando la primera de estas reglas que se ajuste a la situación:

  1. Si cualquiera de los operandos es double, el otro operando se convierte en double.
  2. Si alguno de los operandos es float, el otro operando se convierte en float.
  3. Si cualquiera de los operandos es long, el otro operando se convierte en long.
  4. Si cualquiera de los operandos es un int, el otro operando se promueve a un int.
  5. Si ninguno de los operandos es double, float, long o int, ambos operandos son ascendido a int.

No hay ninguna conversión implícita de tipo numérico a char o booleano. En Java, los tipos char y boolean no son compatibles.

Ejemplos:

   int numeroEntero = 5;
   double v1 = numeroEntero;         // Conversión automática de int a double
   double v2 = 25.36958 / numeroEntero   // numeroEntero se convierte a double antes de realizarse la operación.

Conversión explícita

Se utiliza cuando se desea convertir un tipo de datos en otro tipo de datos que no es compatible automáticamente. Esto requiere que el programador indique explícitamente la conversión utilizando paréntesis y el tipo de datos al que se desea convertir.

   double numeroDecimal = 3.14;
   int numeroEntero = (int) numeroDecimal; // Conversión explícita de double a int

En este caso, estamos realizando una conversión explícita de numeroDecimal a numeroEntero. Sin embargo, es importante tener en cuenta que esta conversión implica una pérdida de precisión, ya que se descartan los decimales. Es importante tener cuidado al realizar casting explícito, ya que puede conducir a resultados inesperados o pérdida de datos. Siempre verifica si la conversión es segura antes de realizarla.

Clases de envoltura (Wrapper classes)

Las clases de envoltura en Java proporcionan métodos para convertir entre tipos primitivos y objetos. Por ejemplo:

  String numeroTexto = "123";
  int numero = Integer.parseInt(numeroTexto); // numero -> 123
  double numDoble = Double.parseDouble(numeroTexto);  //numDouble -> 123.0   
  float numDoble = Float.parseFloat(numeroTexto);     //numFloat -> 123.0

Integer.parseInt ("string numérico") -> convierte una cadena numérica en un int.

Double.parseDouble("string numérico") -> convierte una cadena numérica en un double.

Métodos de conversión de clases de envoltura

Las clases de envoltura también proporcionan métodos como valueOf() para convertir tipos primitivos a objetos y toString() para convertir objetos a cadenas. Por ejemplo:

   int numero = 42;
   double numDoble = 36.45821; 
   String numeroTexto;
   numeroTexto = Integer.toString(numero);          // numeroTexto -> "42"
   numeroTexto = Double.toString(numDoble);         // numeroTexto -> "36.45821"
   numeroTexto = String.valueOf(numero);            // numeroTexto -> "42"
   numeroTexto = String.valueOf(numDoble);          // numeroTexto -> "36.45821"

Conversión entre tipos numéricos

Puedes realizar conversiones entre diferentes tipos numéricos utilizando métodos como intValue(), doubleValue(), longValue(), etc., disponibles en las clases de envoltura. Por ejemplo:

   double numeroDouble = 3.14;
   int numeroEntero = Double.valueOf(numeroDouble).intValue(); // Convierte double a int
